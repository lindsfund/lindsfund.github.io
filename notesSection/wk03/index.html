Mobile UX Design Principles

Basic usability principles

  • Learnability:
    how easily/quickly basics are learned by new users
  • Efficiency:
    once learned, how quickly users perform basic tasks
  • Memorability:
    can users remember how to use the system
  • Errors:
    # of errors and how quickly they can be corrected
  • Satisfaction:
    a positive overall impression of the design/use

    • 1. Content Prioritization

    Less is more. Display only essential content. Secondary content should be available via menu and menu should be hierarchy based. Use icons instead of text where possible.

    2. Make Navigation intuitive

    Pathways for navigation should be clear to the user. Don't make users navigate between apps and pages for a single task.

    3. Touchscreen Target Sizes

    Make interfaces big enough for finger interaction. Apple's suggested minimum is 44px X 44px, Microsoft recommends 9mm/34px. Spacing in between is important as well, especially when it comes to contradicting action buttons. Include an undo button if possible.

    4. Provide User Control

    Allow for user personalization. ie..changing settings, controlling notifications, canceling actions. Inform the user of actions where possible. Allow for demos (guest sign in) before commitment.

    5. Legible Text Content

    "The key to mobile typography is readability." Smaller than 16px becomes difficult to read. Overly large can cause awkward breaks in the page. 30-40 characters per line for mobile screens. Adding space between text can keep users from being overwhelmed.

    6. Make Interface Elements clearly visible

    Contrast between background and elements should be enough to maintain visibility in all different surroundings. Test it out to make sure. WC3 Accessibility Guidelines

    7. Hand position controls

    photo referance
    49% of people rely on one thumb to navigate phone. common features should be placed in easy to reach quadrents. Keep in mind ALL different kinds of users.

    8. Minimize Data Input

    Minimize typeing with use of 'remember me' options, drop downs, location detection, auto-complete, etc. Display keyboard variations based on what info is needed. (number pad for phone number etc.)

    9. Create a seamless experience

    Reduce friction, minimize steps, and make use of devices' features. The experiance should flow between all differnt devices/screens. Sync information across devices as well. Communication with users should be part of this.

    10. Test Your Design

    "Test early. Test often." Build projects with a user centered approach. The earlier you discover problems the quicker and easier it can be to fix them.
